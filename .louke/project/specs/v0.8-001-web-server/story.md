# v0.8-001-web-server — Story

给 `louke` 增加一个面向团队内部使用的 web server，让用户可以直接在浏览器里完成核心协作，而不必只依赖本地文件和命令行。

本期用户故事：

1. 作为团队成员，我希望在 web 页面中通过拖拽方式给 Agent 和抽象角色绑定模型，这样我可以直观看到当前模型编排关系，并快速调整。
2. 作为团队成员，我希望在 web 页面中阅读和编辑 wiki，这样知识沉淀不再只能通过本地 Markdown 文件完成。
3. 作为团队成员，我希望在 web 页面中阅读和修改设计文档，至少覆盖 `spec.md`、`acceptance.md`、`test-plan.md`，这样需求澄清、验收约束和测试设计都能在同一个界面里协作。
4. 作为需求参与者，我希望设计文档不仅能编辑正文，还能良好支持 `inline-discussion`，这样人与 Agent 的澄清可以继续沿用 louke 既有协议，而不是退化成脱离上下文的聊天窗口。
5. 作为文档阅读者，我希望每个 FR / NFR 都能以卡片方式渲染，这样我可以更快浏览状态、测试性和决策状态。
6. 作为文档阅读者，我希望设计文档在渲染时，支持常用的 Markdown 格式，包括代码块、表格、列表等。
7. 作为文档阅读者，我希望 inline-discussion 在显示时：
   1. 默认使用较淡的颜色，但有按钮切换 看正文、看讨论 -- 实现颜色对换
   2. 折叠讨论

已明确边界：

- 模型绑定需要同时支持“按具体 Agent 绑定”和“按抽象角色绑定”。
- wiki 需要支持编辑，不只是只读浏览。
- 设计文档范围至少包括 `spec.md`、`acceptance.md`、`test-plan.md`。
- 面向团队内多人使用，不是单人本地玩具页面。
- 具体技术栈、前后端架构、部署方式、并发编辑实现细节留给 `M-ARCH` 的 Archer 决定。
