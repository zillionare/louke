# v0.8-001 — Web Server 协作界面 — Acceptance Criteria

- **Spec ID**: v0.8-001-web-server
- **创建日期**: 2026-07-08

> 验收标准集中处。`spec.md` 只保留 FR / NFR 的需求描述与元数据；这里定义可观察、可断言的通过条件。
> **编号约定**：每个 `### AC-N` 在机械追踪时规范化为 `AC-FRXXXX-NN` 或 `AC-NFRXXXX-NN`；例如 `FR-0100` 下的 `AC-1` 等价于 `AC-FR0100-01`。

## FR-0100 团队内多人可访问的 web server 工作台

### AC-1
Canonical ID: `AC-FR0100-01`
- 仓库提供明确的 web server 启动入口，启动后浏览器可访问工作台首页。
- 首页同时提供“模型绑定”、“wiki”、“设计文档”三类入口，而不是分散为三个互不相干的 demo 页面。

### AC-2
Canonical ID: `AC-FR0100-02`
- 在同一服务实例下，至少两个不同浏览器会话可访问同一工作台。
- 第二个会话不需要修改源码或重启服务即可看到现有工作区内容。

## FR-0200 拖拽式模型绑定

### AC-1
Canonical ID: `AC-FR0200-01`
- 模型绑定页面存在可拖拽的模型项和可接收绑定的目标项。
- 用户通过拖拽完成一次绑定后，页面能立即显示新关系。

### AC-2
Canonical ID: `AC-FR0200-02`
- 用户可以通过可视化操作取消或改绑已有关系。
- 绑定关系变化后，刷新页面仍能看到最新结果。

## FR-0300 同时支持 Agent 绑定与抽象角色绑定

### AC-1
Canonical ID: `AC-FR0300-01`
- 绑定页面中同时可见具体 Agent 目标和抽象角色目标。
- 用户可以分别对这两类目标执行绑定操作。

### AC-2
Canonical ID: `AC-FR0300-02`
- 当某个 Agent 既受角色默认绑定影响又存在显式绑定时，界面能区分这两种来源。
- 用户能在页面上看出当前生效关系不是单层扁平映射。

## FR-0400 wiki 浏览与编辑

### AC-1
Canonical ID: `AC-FR0400-01`
- 用户可以在 web 页面中查看 wiki 列表并打开任一 wiki 页面。
- 打开后能看到页面正文，而不是只看到文件名或原始目录结构。

### AC-2
Canonical ID: `AC-FR0400-02`
- 用户修改 wiki 内容并保存后，刷新页面仍能看到保存结果。
- 保存后的内容继续保留 Markdown 文本，而不是被转换成不可逆的私有格式。

## FR-0500 设计文档浏览与编辑

### AC-1
Canonical ID: `AC-FR0500-01`
- web 页面可打开 `spec.md`、`acceptance.md`、`test-plan.md` 三类文档。
- 三类文档都支持阅读，不存在其中某类只能下载不能在线查看。

### AC-2
Canonical ID: `AC-FR0500-02`
- 用户可在页面内编辑上述文档并保存。
- 保存后原文件中的标题、锚点、FR / NFR 编号和列表结构未被破坏。

## FR-0600 `inline-discussion` 渲染与编辑支持

### AC-1
Canonical ID: `AC-FR0600-01`
- 当设计文档中存在 `inline-discussion` 线程时，页面能在接近原文上下文的位置展示线程内容。
- 用户能区分发起评论、回复和状态信息，而不是只看到一段未结构化的纯文本。

### AC-2
Canonical ID: `AC-FR0600-02`
- 用户可以在页面内发起新讨论或回复已有讨论，并保存回 Markdown 文件。
- 保存后生成的文本仍兼容现有 `inline-discussion` 协议和工具读取。

## FR-0700 FR / NFR 卡片式渲染

### AC-1
Canonical ID: `AC-FR0700-01`
- spec 页面中每个 FR / NFR 都有卡片化视图，不只是整页纯 Markdown 文本。
- 卡片至少展示编号、标题和正文摘要。

### AC-2
Canonical ID: `AC-FR0700-02`
- 卡片中可见 `Valid`、`Testable`、`Decided` 等元信息。
- 用户可以从卡片跳转或定位回原始文档中的对应段落。

## FR-0800 协作可见性与变更提示

### AC-1
Canonical ID: `AC-FR0800-01`
- 文档或绑定关系变化后，界面会提示当前内容已更新，避免用户在旧视图上继续操作。
- 页面存在“最后修改时间”或“最后修改人”等等价可见信息。

### AC-2
Canonical ID: `AC-FR0800-02`
- 当两个会话对同一内容连续修改时，系统不会静默吞掉其中一次变化。
- 至少有一种可观察的冲突提示、覆盖提示或重新加载提示。

## FR-0900 常用 Markdown 格式渲染支持

### AC-1
Canonical ID: `AC-FR0900-01`
- 设计文档和 wiki 页面中的代码块在 web 页面中以代码块形式展示，而不是退化为普通段落文本。
- 代码块中的换行和缩进在渲染后保持可辨识。

### AC-2
Canonical ID: `AC-FR0900-02`
- 设计文档和 wiki 页面中的表格在 web 页面中以表格结构展示。
- 设计文档和 wiki 页面中的列表在 web 页面中保留有序/无序层级，而不是被扁平化为纯文本。

### AC-3
Canonical ID: `AC-FR0900-03`
- 设计文档与 wiki 页面采用双栏工作台：左侧为 Markdown 源码编辑区，右侧为实时预览区。
- 用户在左侧修改代码块、表格或列表后，可在右侧立即看到对应的结构化渲染结果，而不需要整页跳转或切换到另一条独立路由。

## FR-1000 `inline-discussion` 的正文/讨论聚焦与折叠

### AC-1
Canonical ID: `AC-FR1000-01`
- 含 `inline-discussion` 的设计文档默认以较弱的视觉样式展示讨论线程。
- 默认状态下，正文仍然是主要阅读对象。

### AC-2
Canonical ID: `AC-FR1000-02`
- 页面提供“看正文 / 看讨论”之类的明确切换入口。
- 切换后，正文与讨论的视觉主次会发生对换，而不是只有一个无反馈的开关。

### AC-3
Canonical ID: `AC-FR1000-03`
- 用户可以折叠某个讨论线程或当前文档中的讨论区域。
- 折叠后正文阅读区域变得更聚焦，同时讨论内容可被再次展开。

## NFR-0100 Markdown 兼容与工作流保真

### AC-1
Canonical ID: `AC-NFR0100-01`
- 通过 web 页面保存后的 `spec.md`、`acceptance.md`、`test-plan.md`、wiki 页面，仍可被仓库中的现有 CLI / 解析逻辑读取。
- 不需要为 web 产物额外引入第二套专用文件格式。

## NFR-0200 交互稳定，不得依赖导致明显闪烁的粗暴刷新

### AC-1
Canonical ID: `AC-NFR0200-01`
- 执行模型绑定、文档切换、保存后，页面不会通过整页重载来完成核心交互。
- 用户可连续操作而不会遭遇明显闪烁或焦点频繁丢失。

## NFR-0300 面向团队内使用的可部署性

### AC-1
Canonical ID: `AC-NFR0300-01`
- 交付物包含可运行的服务实现，而不是仅静态原型稿。
- 团队成员按文档启动后，可在浏览器中访问并使用核心能力。
