## 功能需求

> **格式约定（必读）**: 每个 FR 单元由三级标题 + 空格 + FR-XXX (大写, 3 位零填充) + {title} 引起, 紧接一个 3 列表格存放元数据 (有效需求/可测性/是否已决定), 然后是需求描述; FR 之间用 `---` 分隔.
>
> **编号约定(必读)**: FR 的编码采用 3 位数字, 0 填充, 初稿时从 10 开始, 每次增加 10; 以便后续可以随时在中间插入新的 FR.
>
> **必读**: FR-XXX 编号即该需求的 id. 禁止删除已有需求 id, 以避免引用混淆; 如需废弃某 FR, 在表格里把 `有效需求` 改为 `❌`, 并在澄清记录中说明.
>
> **元数据字段 (表格列)**:
> - 有效需求 (原 yaml `valid`): `✅` = 仍生效, `❌` = 已废弃
> - 可测性 (原 yaml `testability`): `✅` = 可被测试断言, `⚠️ {原因}` = 有保留
> - 是否已决定 (原 yaml `resolved`): `✅` = 用户已 review 通过, `⚠️` = 待澄清, `❌` = 用户明确否决
