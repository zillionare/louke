# v0.12 Real OpenCode L3 Smoke Runbook

This is the **honest** L3 smoke for v0.12 (test-plan §5.3, AC-FR1401-01,
AC-NFR0301-04). It exercises a real `opencode serve` process via
`louke.opencode.real.RealOpenCodeAdapter` — **no mock, no echo**.

> The previous `tests/e2e/test_real_opencode_l3__smoke.py` was deleted in
> v0.12 because it actually exercised `InMemoryOpenCodeAdapter` through a
> Starlette `TestClient`, so its `any(marker in m["content"] ...)` assertion
> was a tautology (the in-memory adapter echoes the marker directly).
> See issue #170 for the replacement and #161 for the original (closed) issue.

## Test file

`tests/e2e/test_real_opencode_l3_e2e.py`

Marker: `@pytest.mark.real_opencode` (registered in `tests/conftest.py`).

## What it does

1. `adapter.create(correlation_id=...)` → real `POST /session` on the
   `opencode serve` process.
2. `adapter.send_message(instance_id, "echo <marker>", ...)` → real
   `POST /session/{id}/prompt_async` (returns 204 accepted).
3. Polls `adapter.list_messages(instance_id, ...)` → real
   `GET /session/{id}/message` until an assistant message containing the
   marker appears (timeout 15s).
4. `adapter.stop(instance_id)` → real `DELETE /session/{id}` in a `finally`
   block, so the session is torn down even on assertion failure.

Failures are reported as-is (HTTP status + body snippet via
`RealOpenCodeAdapter._request` / `_snippet`). No retries that hide failures.

## Prerequisites

- A real `opencode serve` process reachable over HTTP. Example:

  ```bash
  opencode serve --port 41234
  # wait until http://127.0.0.1:41234/global/health returns 200
  ```

- No `OPENCODE_API_KEY` is required by the adapter. The adapter talks to
  the opencode server HTTP API only (`POST /session`, `POST .../prompt_async`,
  `GET .../message`, `DELETE /session/{id}`, `GET /global/health`). Provider
  credentials, if any, are configured inside `opencode serve` itself — the
  Louke test process does not handle or read them.

## Run

```bash
export LOUKE_OPENCODE_BASE_URL=http://127.0.0.1:41234
export LOUKE_RUN_REAL_OPENCODE=1
.venv/bin/python -m pytest -m real_opencode tests/e2e/test_real_opencode_l3_e2e.py -v
```

## Skip behaviour (this is correct, not a bug)

Without `LOUKE_RUN_REAL_OPENCODE=1` **and** a reachable
`LOUKE_OPENCODE_BASE_URL`, the test is **skipped**, never passed:

```
$ .venv/bin/python -m pytest tests/e2e/test_real_opencode_l3_e2e.py -q
s                                                                        [100%]
1 skipped in 0.03s
```

A health-check failure (server unreachable) also skips, never errors:

```
$ LOUKE_RUN_REAL_OPENCODE=1 LOUKE_OPENCODE_BASE_URL=http://127.0.0.1:1 \
    .venv/bin/python -m pytest tests/e2e/test_real_opencode_l3_e2e.py -q
s                                                                        [100%]
1 skipped in 0.05s
```

Per test-plan §1.3, this is an environment-gated skip (real-provider
availability), not a silent skip that hides a missing test.

## CI

This test is **not** part of the default CI green path. It is a release /
nightly gate. Per test-plan §6.4: if the provider is unavailable, the L3
must report a clear unavailable/error, never `completed` — this test does
that by skipping with an attached reason rather than faking a pass.
