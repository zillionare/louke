# v0.12 Real OpenCode L3 Smoke Runbook (proven path)

This is the **honest** L3 smoke for v0.12 (test-plan §5.3, AC-FR1401-01,
AC-NFR0301-04). It exercises a real `opencode serve` process via
`louke.opencode.real.RealOpenCodeAdapter` — **no mock, no echo**.

> The previous `tests/e2e/test_real_opencode_l3__smoke.py` was deleted in
> v0.12 because it actually exercised `InMemoryOpenCodeAdapter` through a
> Starlette `TestClient`, so its `any(marker in m["content"] ...)` assertion
> was a tautology. See issue #170 for the replacement.
>
> Initial `RealOpenCodeAdapter` (B4) shipped with **wrong HTTP paths**
> (`/prompt_async`, `/session/{id}/message?limit=N`) — it returned 200 OK
> but never reached an LLM. The bug was fixed in commits `e8f6a12` and
> `c7b4677`; the corrected paths are documented below. The earlier v0.12.0
> tag was force-advanced despite this and removed in the v0.12 audit.

## Tests

1. **e2e L3 smoke** — `tests/e2e/test_real_opencode_l3_e2e.py`
   Marker: `@pytest.mark.real_opencode` (registered in `tests/conftest.py`).
2. **Unit live integration** — `tests/unit/opencode/test_real.py`

   - Both skip by default; gated on `OPENCODE_INTEGRATION=1` + `OPENCODE_LIVE_BASE_URL`.

## What the adapter talks to (verified against opencode 1.17.15)

| Method | Path | Body | Returns |
|---|---|---|---|
| `adapter.create()` | `POST /api/session` | `{"directory": "/tmp", "model": {"providerID": "opencode", "id": "big-pickle"}}` | `{data: {id, time, model, ...}}`; unwrap envelope → `id` |
| `adapter.list()` | `GET /api/session` | — | `{data: [{id, ...}]}`; flatten array |
| `adapter.send_message()` | `POST /api/session/{id}/prompt` | `{"prompt": {"text": "..."}}` | `{data: {id, prompt, delivery, ...}}`; admitted message id |
| `adapter.list_messages()` | `GET /api/session/{id}/message` | — | `{data: [{id, type, content: [{type, text}], ...}]}`; `role=m["type"]`, `text="".join(p["text"] for p in m["content"] if p["type"]=="text")` |
| `adapter.stop()` | `DELETE /api/session/{id}` | — | `true` |
| `adapter.cancel()` | `POST /api/session/{id}/abort` (falls back to `stop()` on 404) | — | — |
| (health probe) | `GET /global/health` | — | `{healthy: true, version: "1.17.15"}` |

The previous (broken) paths — `POST /session/{id}/prompt_async`,
`GET /session/{id}/message?limit=N` — are gone.

## Prerequisites

1. **A real `opencode serve` process reachable over HTTP.** Example:

   ```bash
   # Use a config that exposes the FREE `opencode/big-pickle` model (no API key).
   cat > /tmp/opencode_cfg.json <<EOF
   {
     "\$schema": "https://opencode.ai/config.json",
     "model": "opencode/big-pickle",
     "provider": {
       "opencode": {
         "npm": "@ai-sdk/openai-compatible",
         "name": "opencode",
         "options": {"baseURL": "https://opencode.ai/zen/v1"},
         "models": {"big-pickle": {"name": "Big Pickle"}}
       }
     }
   }
   EOF
   OPENCODE_CONFIG=/tmp/opencode_cfg.json opencode serve --port 41234

   # wait until http://127.0.0.1:41234/global/health returns 200
   ```

2. **No API key required.** The adapter talks to the opencode server HTTP API
   only; provider credentials are configured inside `opencode serve` itself.
   `opencode/big-pickle` is a free model (cost.input = cost.output = 0) reachable
   at `https://opencode.ai/zen/v1` — verified by running
   `opencode run --model opencode/big-pickle "echo SMOKE"` and getting the
   echo back.

## Run

### e2e L3 smoke

```bash
export LOUKE_OPENCODE_BASE_URL=http://127.0.0.1:41234
export LOUKE_RUN_REAL_OPENCODE=1
.venv/bin/python -m pytest -m real_opencode tests/e2e/test_real_opencode_l3_e2e.py -v
```

Expected: `1 passed` (the assistant echoes the smoke marker within 15s).

### Unit live integration tests (faster, in-process)

```bash
export OPENCODE_INTEGRATION=1
export OPENCODE_LIVE_BASE_URL=http://127.0.0.1:41234
.venv/bin/python -m pytest tests/unit/opencode/test_real.py -v -k "live"
```

Expected: `2 passed` (status probe + create/send/list/delete).

## Skip behaviour (correct, not a bug)

Without the env vars + reachable server, the tests **skip**, never pass:

```
$ .venv/bin/python -m pytest tests/e2e/test_real_opencode_l3_e2e.py -q
s                                                                        [100%]
1 skipped in 0.03s
```

A health-check failure (server unreachable) also skips, never errors.

Per test-plan §1.3, this is environment-gated skip (real-provider availability),
not a silent skip that hides a missing test.
