# v0.12.0 M-MILESTONE — L3 Real-OpenCode Smoke Runbook

This runbook describes how to run the L3 real-OpenCode smoke test
(`tests/e2e/test_real_opencode_l3__smoke.py`) for the v0.12.0 release.

## Pre-requisites

1. **Real OpenCode credentials** — set one of:
   - `OPENCODE_API_KEY=<your-key>`, or
   - `OPENCODE_BASE_URL=<your-endpoint>` (for self-hosted providers)

2. **Permission to run real-OpenCode tests** — explicit per
   `meta.security_audit = "disabled"` in `.louke/project/project.toml`:
   the project DoD explicitly requires "真实 OpenCode smoke 有明确证据"
   but the audit *stage* is skipped. Run only in a controlled environment.

3. **Clean git fixture** — work in a throwaway branch, not on `main`.

## Run the smoke

```bash
# 1. Activate the project venv
source .venv/bin/activate

# 2. Verify the L3 marker is registered
pytest --markers | grep real_opencode
# expected: real_opencode: L3 smoke test that requires a live OpenCode ...

# 3. Run the L3 smoke (must set LOUKE_RUN_REAL_OPENCODE=1 to actually run)
LOUKE_RUN_REAL_OPENCODE=1 \
OPENCODE_API_KEY=$OPENCODE_API_KEY \
pytest -m real_opencode tests/e2e/test_real_opencode_l3__smoke.py -v
```

Expected output: `1 passed` plus a tear-down trace (no instance left behind).

## Honesty rules

Per `test-plan.md §5.3` and `AC-NFR0301-01`:

- The test **must not** be modified to fake a pass.
- A real OpenCode failure **must** be reported as-is.
- The test **must not** be replaced with a mock adapter for the L3 path.
  Mock adapters belong in L1/L2 (covered by `tests/e2e/test_opencode_e2e.py`
  via `Starlette TestClient`).

## Reporting

After the run, capture:

1. **Pass/fail** of `test_real_opencode_create_attach_send_detach_end_l3_smoke`.
2. **OpenCode session evidence** — `server_id`, `workspace_id`, `session_id`
   returned by the adapter.
3. **Provider metadata** — model id, response latency, exit code.
4. **Teardown** — confirm the instance was released by the `finally` block.

Attach the captured report to the issue #161 close comment and reference it
in any future v0.12.x release notes.
